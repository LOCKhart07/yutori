v20 — AI rule entry point on Recipient rules (#156)

The describe-a-rule sheet (shipped in #64 part 2) currently only appears on unsure suggestion cards — gated on !confident && onDescribeThisRule != null inside SuggestionCard. A user with no unsure suggestions can enable AI in settings, download the 2.58 GB model, and still have no way to use the feature. Issue #151 (closed as dup of #64) is exactly that.

Fix: promote the describe-sheet entry to a permanent button on the Recipient rules screen, alongside the existing “+ Add a new rule”. The AI-settings screen grows a “Try it” row (after the model is installed) that navigates to Recipient rules — not a second launcher. Single source of truth for the feature’s entry point.

B2 · New “Try it” row. Only shown when enabled && modelInstalled. Tapping navigates to Recipient rules — not a second describe-sheet launcher. The user then sees the new top-level entry from A2 and taps it there.
Cold-start path when AI is off or model missing. If the user taps “Describe a rule in plain English” (A2) without aiReady, reuse the existing AiUnavailableSheet — same component that already guards the unsure-card path today. The sheet explains that AI is off / model isn’t installed and links to AI settings. No new guard surface.
Why “Try it” navigates, not launches. If the settings screen directly opened the describe sheet, we’d have two independent launchers for the same feature — easy for them to drift (different seed defaults, different unavailable-state handling, divergent analytics). Navigation keeps Recipient rules as the single home and teaches the user where the feature lives. “Try it” is a signpost, not a duplicate.
Ordering: describe-a-rule sits above Add-a-new-rule. AI path is higher-intent and lower-effort (describe in English vs. fill out a form). Putting it first matches the screen’s progression from suggested → assisted → manual.
Empty-state affordance under Suggested header — dropped. Option 3 in the issue was a per-empty-state sentence under “SUGGESTED” when the list is empty and AI is ready. With A2 the top-level entry already solves discoverability; adding a second sentence would be chrome. Skip.
Not in scope. No changes to the describe sheet itself, AiUnavailableSheet, the extraction pipeline, or the post-extraction AddEditRecipientRule pre-fill. Wiring only.
