v7 · Historical-import landing affordance — issue #74

When the user runs Import past SMSes, rows almost always land in past months (Jan/Feb/Mar) because that's where the source SMS dates point. Today the dashboard just says Imported 47 new, skipped 2 duplicates and stays on the current month — the user sees no new rows and often believes the import did nothing. The affordance below tells them where the rows landed and jumps them there. Picking one of A/B/C drives the ImportStatus.Succeeded shape: A needs earliest-month + total, B needs the month range + count, C needs per-month-direction counts.

Recommendation

A as the primary affordance. It's the least risky shape, matches the existing import-status-line style, and needs the smallest ImportStatus.Succeeded change (one nullable earliestMonthTouched field). If discoverability on import is under-performing in practice, promote to B in a follow-up — don't lead with the heavier surface. C is tempting but the chevron is already loaded (the pager swipe gesture lives there) and a badge risks misreading as an error dot.
