v18 — bury the ingestion log off the dashboard (#146)

The "Latest ingested messages" block on the dashboard is distracting — it's debug-flavoured chrome sitting next to the month's spend total. Move it to a dedicated Message log screen reachable from Settings → Data & accounts. The dashboard loses the block entirely; the data isn't lost, just one tap away for when the user wants to verify "did my last SMS classify correctly?". No change to ingestion, parsing, or data model — purely a UI relocation.

A — Dashboard: before vs. after

The current block shows 5 rows — each is a sender + body preview + outcome pill. That's four lines of type-dense information directly under the spend total. Removing it lets the month number and Today's transactions breathe.

B3 · empty state. Only reachable right after a fresh install — once you've imported history or let SMS flow in, the list has content.
Scope kept tight. No filter chips (All / Affects budget / Needs review / Ignored), no search, no per-row tap-to-open detail sheet, no "re-classify" action. Those are reasonable next steps but out of scope for #146 — the issue asks for burial, not a redesign. If we ship this and the user then asks "can I filter to only Needs review?", that's a follow-up.
Why 50? The DAO today exposes observeLatest(limit); the dashboard block used 5. For the burial screen I'd default to 50 with a "Load more" footer (not drawn). Infinite scroll of 1,200+ rows would be simple to implement but the value drops fast past the last couple of days — the user's mental question is "did my recent SMS get classified?", not "what happened in February?".
Naming — Message log vs alternatives. I picked "Message log" because it's plain, not technical, and reads as reference material. Alternatives: Ingestion log (matches the internal term but leaks jargon), Parsed SMS (accurate but exposes implementation), Message history (fine, slightly more formal). Easy to flip before writing code if you prefer one of these.
No dashboard shortcut. Considered a small icon in the dashboard's top chrome that opens the Message log directly. Left it out — the whole point of the issue is that this is noise on the dashboard; putting a tiny icon back is the thin-end-of-the-wedge. Settings is the right and only home.
