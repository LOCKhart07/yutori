v11 — Unicode-glyph-as-icon sweep (#87 widened)

Every "before" column below is exactly what the Kotlin source says today — a Text(text = "<single Unicode char>") being used as an icon. On your OxygenOS device, font fallback maps several of these to unrelated glyphs (✓ instead of ⟲, ! instead of ◉) which is the bug filed in #87. The fix swaps each to a Compose Icons.* from the Material core set already on the classpath — no new dependency, no custom drawables.

Icons rendered here come from Google's Material Icons web font — same glyph source as Compose's androidx.compose.material.icons. So "After" is a faithful preview of what ships, not an approximation.
