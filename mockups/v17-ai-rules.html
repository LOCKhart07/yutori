v17 — AI-assisted rule creation (#64 part 2, Stage B)

Companion to plans/ai-rules-spec.md. The feature is opt-in via a Settings toggle; on first enable the user downloads a ~2.58 GB on-device model (Gemma 4 E2B via LiteRT-LM, shipped as "Deep"). Stage A confirmed that narrower scope (extract pattern and category, not classification) + a strict JSON output contract scores 8–10/10 on a 10-prompt fixture at the Deep tier. Classification stays with the user's manual dropdown pick in AddEditRecipientRule (#28). No weights ship in the APK. No cloud call. No token.

A — Settings → AI-assisted rules · lifecycle

New section on the Settings screen, directly under Recipient rules (#28 surface). The toggle is off by default; tapping it opens a confirmation sheet that states the download size + network recommendation + privacy posture before we commit the user to anything.

C — The Describe-this-rule sheet · four states

Modal bottom sheet. Single-purpose. Free-text in, extracted draft out. Inference takes ~10 s on the Nord's Snapdragon 7+ Gen 3; the sheet shows a spinner, the text field stays visible, and the primary button disables. On success the sheet dismisses and the user lands on #28's edit form with the extracted values pre-filled (screen D).

What's deliberately missing. No onboarding nag, no home-screen "Try AI rules" banner, no first-run promotion. Discovery happens only where the user is already trying to write a rule. If the user never hits an unsure suggestion and never uses #29's per-tx menu, they never see the feature — which is fine. The Part-1 heuristic path already covers the common case.
No v1 picker. The Settings card shows only "Deep". Multi-tier ("Quick" 270M / "Balanced" 1B) is deferred pending non-gated smaller Gemma 4 variants — Gemma 3 270M/1B are license-gated on HF and would require a browser-and-file-picker detour per tier that we're not willing to ship. The tier infrastructure (download flow, atomic swap, per-model SHA pin) is still designed tier-agnostic so a future "Change model" row can be added without churn.
