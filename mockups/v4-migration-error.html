Yutori — DB migration failure recovery (#9)

Full-screen recovery state per plans/error-states-spec.md §5.1. Shown when Room.openHelper.writableDatabase throws at app start — typically a missing migration, mismatched schema, or SQLiteDatabaseCorruptException (§5.5). App refuses to enter normal UI; this is the only surface the user sees. Two actions: Copy error details (safe — to clipboard for a bug report) and Clear app data (destructive — wipes Room DB + settings; SMS inbox untouched; first launch after is onboarding). Destructive action is behind a confirm dialog.
