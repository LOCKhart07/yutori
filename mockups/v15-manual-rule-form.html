Behaviours & validation (summary)

  • Pattern. Non-empty; trimmed on save. Monospace input.
  • Pattern kind. Literal default. Regex triggers the Test panel + inline compile-check. Prefix = String.startsWith, case-sensitive.
  • Reclassify as. Required. Options: CC_BILL_PAYMENT, SELF_TRANSFER, REFUND, INCOMING_CREDIT, NON_FINANCIAL (plan §5 classifications that are rule-addressable).
  • Linked account. Only shown when Reclassify-as = SELF_TRANSFER. Required for that target. Empty accounts list → helper nudge to register one.
  • Duplicates. (pattern, pattern_kind) is the DB unique index — save shows an inline "A rule with this pattern already exists" and points to the existing row.
  • Test panel source. SELECT merchant FROM transactions WHERE classification = 'UPI_PAYMENT' AND merchant IS NOT NULL ORDER BY occurred_at_ms DESC LIMIT 100. First 20 hits render.
  • Save. Insert (or update) recipient_rules with source = USER; when prefill came from a suggestion, delete the matching rule_suggestions row. No retroactive reparse (same posture as v14).
  • Edit mode. Read DAO by id, prefill; on Save issue an UPDATE. Delete button only for source = USER | LEARNED. Seed rules get a read-only variant.
