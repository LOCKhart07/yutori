v10 · Settings → Send feedback (#113)

In-app quick-capture of bugs and improvement ideas. Today's scope is single-installer self-service — you tap, type, send, and the app POSTs to /repos/LOCKhart07/yutori/issues via a fine-grained PAT baked into the APK (secret ISSUES_TOKEN, mirroring the autoupdater's RELEASES_TOKEN pattern).

Scope note: Four frames below cover Settings entry, compose (filled + focused), success snackbar with issue URL, and an error snackbar with retry. Empty-state compose (blank fields, Send disabled) isn't drawn — the only difference is the Send action goes grey until the title field is non-empty.

Lives under a Feedback section header. Placed above the existing App updates section so it's findable without scrolling. Tapping opens the compose sheet (frame B).

Open decisions I want to hear on before coding

  • Copy for the Settings row subtitle. Mockup shows "Report a bug or suggest an improvement." Alternatives: "Bugs & ideas," "Contact the developer," "Help me improve Yutori."
  • Full-screen modal vs bottom sheet. Drawn as full-screen because the description field wants vertical space alongside the keyboard. Bottom sheet feels lighter for "just a quick note." Either works in Compose — happy to switch if you prefer.
  • Context block — show or hide by default. Currently shown inline as a monospace card so you know what's attached. Alternative: collapse behind a "Show details" disclosure like the migration error screen.
