Yutori — v5 mockup: swipe between months on dashboard (#21)

Goal: replace the feeling of "tap, wait, tap" with a horizontal swipe on the dashboard. Chevrons stay (edge discoverability, accessibility), but the gesture is the primary mode. Each month is a full dashboard page — strip, categories, banners, everything.

Decisions locked:
  • Impl: HorizontalPager (Compose foundation). Each month = one page composable.
  • Swipe region: whole dashboard scroll area. Vertical scroll inside the page must yield cleanly to horizontal pager drag at the horizontal-dominant gesture threshold.
  • Range: bounded on the past edge at the earliest sms_log month, unbounded forward (you can page into future months to preset a budget before the month begins).
  • Chevrons: kept in the topbar. Past-edge chevron dims when at earliest month. Forward chevron is never disabled.
  • Peek: 12dp of the neighbor month visible at each edge when at rest, so there's a visual hint the surface is horizontally pageable. Achieved via contentPadding on the Pager.
  • State: each page hosts its own MonthSnapshot; pager window keeps ±1 page composed for fling-smoothness. ViewModel keys snapshots by YYYY-MM.

① At-rest states — peek hints neighbor months exist

At rest, a 12dp sliver of the previous month's surface colour is visible on the left edge (and, for any non-earliest month, the next month on the right). The peek is non-interactive — just a visual affordance. The topbar chevrons remain for explicit taps and accessibility.

② Mid-drag state — user is swiping right, March sliding into view

Drag offset is ≈55% of page width. The previous month (March) occupies the left 55%; the current month (April) has translated to the right and is being pushed out. Shadow gradient softens the trailing edge. Release threshold: 50% of page width or a fling velocity above the configured minimum → snap to March. Otherwise settle back to April.

Topbar month label: stays "April" during drag and flips to "March" only on settle. Avoids flicker if the user releases below the snap threshold. Open Q: or cross-fade the month label proportionally to drag offset?

③ Gesture conflict — vertical scroll vs horizontal page

The dashboard scrolls vertically (long category list, banners below). Pager must not hijack vertical drags. Standard resolution: HorizontalPager only commits to a horizontal gesture once the pointer movement has a horizontal-dominant slope (|dx| > |dy|) and has exceeded a small slop (~6dp). Before that, vertical scroll takes the gesture. Once committed, pager keeps the gesture until release.

Edge behaviours to verify at build time

  • Swiping past earliest month rubber-bands and snaps back (no blank page beyond edge).
  • Forward-swiping into a month with no transactions renders the Empty state (CTA variant "Set May budget" / "Set budget for July").
  • Chevron tap still animates as a pager scrollToPage(index), not an instant jump — so chevron and swipe produce the same visual motion.
  • Dashboard pull-to-refresh (if/when added) shares the vertical axis only; no conflict with pager.
  • Month change emits the existing onMonthPrev / onMonthNext callbacks so ViewModel wiring is unchanged — pager calls these on settle.
  • Accessibility: TalkBack users get the chevrons as discrete buttons; swipe gesture is not the only way to navigate.

What this does not ship

  • No vertical swipe (years, settings, etc.) — horizontal only.
  • No hero-only swipe option — the whole dashboard scrolls as a page.
  • No month-picker grid (quick-jump multiple months ahead) — tracked as #91.
