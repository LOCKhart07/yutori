Unify Edit classification + Edit category sheets

Three shapes for #143. Option 1 is the "minimum" — align state/highlight behaviour across the two existing sheets with no visible UI change. Options 2 and 3 merge into one sheet, differing in how the two pickers share the surface.

Today: tapping Edit category or Edit classification opens its own bottom sheet with title + "Currently: automatic/overridden" hint + scrollable picker + Cancel/Save. Both work but selection semantics drift (category starts selected = current; classification starts selected = override-or-null). Opening them back-to-back exposes the drift.

Option 1 — Keep two sheets, align state + highlight only

Minimum scope. No visible change; fixes only the selection-model drift so both sheets start with selected = current-value-shown and highlight the currently-applied row on open. Two action buttons stay separate, two Save commits. Low risk, shortest diff, closes the issue.

Tradeoffs

1. Light — zero visual change, one-commit fix. No merged-sheet win. Good if you're not sold on combining.

2. Tabs — visibly combined, stays compact, one Save commits both. Adds a small segmented control. Entry points still feel natural (the tapped button sets the initial tab).

3. Stacked — no tab switch to learn, but 15 rows is a long scroll and the Save button is below the fold. Feels heavy in hand.

Recommendation: 2. Compactness + one-Save for related edits outweigh the added segmented control. If you'd rather close the issue without touching the UI at all, option 1.
