Yutori — Restricted-settings onboarding helper (#24)

On Android 13+, sideloaded APKs can't request SMS permissions until the user flips Allow restricted settings in App info. The system silently denies the runtime request with no visible dialog — the user taps Grant permissions and nothing appears to happen. Hit on first install 2026-04-16.

Frame A shows the current permission gate (unchanged — step 1 of the onboarding). Frame B is the new restricted-settings guidance state, reached when the launcher's result comes back denied with shouldShowRequestPermissionRationale = false on Android 13+ (i.e. the dialog never appeared). Primary CTA deep-links to Settings.ACTION_APPLICATION_DETAILS_SETTINGS; secondary CTA re-runs the permission launcher so the user can retry after flipping the toggle without leaving the screen.

Detection heuristic is deliberately generous — we don't try to distinguish "restricted by OS" from "user tapped Deny twice". Both states are dead-ends that the app-info detour resolves, so the same guidance applies.
